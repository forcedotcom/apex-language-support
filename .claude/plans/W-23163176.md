# W-23163176 — fix findMissingArtifact capability name mismatch

## Context

- client advertises `experimental.findMissingArtifactHandler`; server advertises `experimental.findMissingArtifactProvider` — name mismatch
- server name `findMissingArtifactProvider` canonical: read in `ConfigurationSummary.ts`, `ApexCapabilitiesManager.ts`, `LCSAdapter.ts`; typed as `ExperimentalCapabilities.findMissingArtifactProvider: FindMissingArtifactCapability`
- client key `findMissingArtifactHandler` defined only in `ApexClientCapabilities.ts` (3 sites: BASE/PRODUCTION/DEVELOPMENT); never read; `ClientCapabilities.experimental` untyped (`LSPAny`) → no compile error, silent mismatch
- fix: rename client key → `findMissingArtifactProvider`; shape (`enabled`, `supportedModes`) already matches `FindMissingArtifactCapability`, no shape change
- standalone bug fix; prereq for capability gating WI (gating reads client cap under canonical name)
- no prod consumer → behavior-neutral; only snapshot tests affected
