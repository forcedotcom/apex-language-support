# W-23163179 — Server-side capability gating for apex/* sends

## Context

- Four `apex/*` methods sent server→client:
  - `apex/workspaceIngestionComplete` — notification from `setIngestionCompleteCallback` in LCSAdapter (~956)
  - `apex/queueStateChanged` — notification from queue-state task in `handleInitialized` (~1636)
  - `apex/findMissingArtifact` — actual send site: `MissingArtifactResolutionService.ts:304` (`connection.sendRequest('apex/findMissingArtifact', safeParams)`). The `CoordinatorPrimaryAssistanceHandler` dispatches to `LSPQueueManager.submitFindMissingArtifactRequest` which handles queueing; it does NOT send to client itself
  - `apex/requestWorkspaceLoad` — notification from `WorkspaceLoadCoordinator.ts` (~97)
- Client advertises support via `experimental.findMissingArtifactProvider` (`ApexClientCapabilities.ts`)
- `LCSAdapter` stores `clientCapabilities` (~1339) but NOT exposed via `LSPConfigurationManager`
- `LSPConfigurationManager` singleton accessible from all packages; exposing client caps enables gating at any send site
- `APEX_METHODS` registry has `capabilityKey` on `findMissingArtifact`; other sends lack explicit gating
- `ClientCapabilities.experimental` is typed `any` in LSP protocol — runtime access must guard against undefined/non-object values

### Per-method backward-compatibility policy

| Method | Policy | Rationale |
|--------|--------|-----------|
| `apex/workspaceIngestionComplete` | default-allow (send if caps undefined) | Legacy clients expect this notification to drive status-bar UX |
| `apex/requestWorkspaceLoad` | default-allow (send if caps undefined) | Legacy clients expect this to trigger workspace loading |
| `apex/findMissingArtifact` | default-allow (send if caps undefined), gate when caps present but key absent | Legacy clients handle this request; only suppress when we KNOW client opted out |
| `apex/queueStateChanged` | default-deny (caps required) | Dev-mode only; no legacy client depends on this |

## Phases

### Phase 1 — Expose client capabilities on LSPConfigurationManager

- Add field `private clientCapabilities?: ClientCapabilities;` after line 80 (`runtimeDependencies` field)
- Add `setClientCapabilities(caps: ClientCapabilities | undefined): void { this.clientCapabilities = caps; }`
- Add `getClientCapabilities(): ClientCapabilities | undefined { return this.clientCapabilities; }`
- Add helper with safe runtime guard:
  ```typescript
  isClientCapabilityAdvertised(capabilityKey: string): boolean {
    const exp = this.clientCapabilities?.experimental;
    return exp != null && typeof exp === 'object' && exp[capabilityKey]?.enabled === true;
  }
  ```
  `experimental` typed `any` in LSP protocol — `typeof exp === 'object'` guard prevents runtime errors from primitives/null
- `LCSAdapter.handleInitialize`: call `configManager.setClientCapabilities(params.capabilities)` after the existing local store at ~1339
- Unit tests:
  - get/set round-trip
  - `isClientCapabilityAdvertised` returns true for present key with `enabled: true`
  - returns false for present key with `enabled: false`
  - returns false for absent key
  - returns false when `experimental` is `undefined`
  - returns false when `experimental` is non-object (e.g., string, number)
  - returns false when `clientCapabilities` itself is `undefined` (not yet set)

### Phase 2 — Gate server-initiated sends

- Guard `setIngestionCompleteCallback` lambda; skip send only if `clientCapabilities` defined AND `isClientCapabilityAdvertised('workspaceIngestionProvider')` false; if `clientCapabilities` undefined (legacy), send unconditionally (default-allow)
- Guard `notificationCallback`; send only if `isClientCapabilityAdvertised('queueStateProvider')` true (default-deny; dev-mode only, no legacy dependency)
- Gate at send site `MissingArtifactResolutionService.resolveInBackground` before `connection.sendRequest('apex/findMissingArtifact', ...)` line 304; check `isClientCapabilityAdvertised('findMissingArtifactProvider')` OR `getClientCapabilities() === undefined` (default-allow legacy); if gated, debug-log and return early
- `WorkspaceLoadCoordinator.sendRequestWorkspaceLoadNotification`: accept capability-check callback; skip send only if `clientCapabilities` defined AND `isClientCapabilityAdvertised('requestWorkspaceLoadProvider')` false; if undefined, send unconditionally (default-allow)
